Your Mac stays awake.
MDM can't touch this.

KeepAwake reads Jamf screensaver policies and power management timers, then simulates invisible user activity at the optimal interval. Auto-pauses when your screen locks, skips ticks while you're active, and backs off on low battery — zero effort, zero footprint.

// the problem

caffeinate doesn't cut it.

Tools like caffeinate, Amphetamine, and KeepingYouAwake only prevent system sleep via IOKit assertions. They do not stop the screensaver lock enforced by Jamf MDM profiles.

Others

Prevent sleep only

IOKit power assertions keep the system running — but the screensaver daemon has its own idle counter. Screen still locks.

KeepAwake

Resets HIDIdleTime

Simulates a real fn keypress via System Events, which resets the OS idle counter that the screensaver daemon actually monitors.

// capabilities

What's under the hood

☍

MDM Policy Detection

Reads Jamf screensaver profiles at /Library/Managed Preferences and parses pmset power timers automatically.

⏱

Smart Intervals

Calculates 80% of the shortest detected timer. Never too early, never too late. Clamped between 10s and 300s.

⚡

Instant Power Switching

IOKit power source callbacks detect AC/battery changes in real time. Interval adjusts instantly — no polling delay.

🔒

Screen Lock Auto-Pause

Detects when your screen is locked and pauses all activity. Releases the display assertion while away, resumes instantly on unlock.

✋

Idle Detection Override

Skips fake keypresses while you're typing or using the mouse. Only simulates activity when you're truly idle — no interference.

🔋

Battery Threshold

Auto-disables when battery drops below your threshold (5–50%). 5% hysteresis prevents rapid toggling. Resumes when charged.

☼

Invisible Activity

Sends fn key (code 63) via CGEvent. No visible effect, no modifier conflict. Just a silent idle reset.

⚙

Native Settings

General, Timing, and About tabs. Configure lock behaviour, battery threshold, idle detection, intervals, and launch preferences.

🛡

Display Sleep Guard

Holds an IOPMAssertion to prevent display sleep independently. Automatically released during screen lock and battery suppression.

// pipeline

Five steps, every tick

01

Detect

Read MDM screensaver profile & pmset sleep timers

02

Calculate

Pick shortest timer, fire at 80% of that value

03

Guard

Check screen lock, battery level, and user activity

04

Simulate

Send fn keypress via CGEvent to reset HIDIdleTime

05

Adapt

IOKit callback detects power change, adjust interval instantly

Install

DMG from GitHub Releases, or brew install --cask yashiels/tap/keepawake. Build from source: make run.

Menu bar

Sun icon when active, moon when paused. Green dot for active, orange for paused (locked or low battery), grey for disabled.

Privacy

Zero analytics. Zero telemetry. No network calls except optional GitHub update checks. Screen lock detection uses local polling — no traces.

Permissions

Accessibility access for CGEvent key simulation. No Full Disk Access. No admin rights needed to read MDM policies.
